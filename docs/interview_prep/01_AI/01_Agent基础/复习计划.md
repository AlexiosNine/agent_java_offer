## 三、每次复习的评分维度

每次复习时，对每道题从两个维度打分（各 5 分，合计 10 分）：

### 维度 A：知识点确认（5分）

> 考察：概念是否清晰、关键组件是否完整、常见数字是否记住

| 分值 | 标准 |
|------|------|
| 5 | 核心概念清晰，组件完整，数字准确，能应对追问 |
| 4 | 核心概念清晰，组件基本完整，数字偶有模糊 |
| 3 | 核心概念清晰，但组件有遗漏，数字模糊 |
| 2 | 概念有偏差，组件不完整 |
| 1 | 概念混乱，需要重新学习 |

**Agent基础 各题知识点要点**：

- **题1 Agent定义**：四层架构（规划/记忆/工具/执行控制）、与普通对话的3个对比维度
- **题2 ReAct**：三步循环（Thought→Action→Observation）、与CoT的区别、动态调整特性
- **题3 工具超时**：四层防御（重试→降级→熔断→转人工）、关键数字（3次/5秒/10次/60秒）
- **题4 产品设计**：四维选场景（高频/可量化/可回滚/人力成本高）、四阶段落地、三类指标
- **题5 记忆系统**：三层记忆（短期/长期/工作）、三种触发时机、存储选型（向量库+关系型）
- **题6 评估体系**：控制变量法（Prompt/RAG/工具）、golden test set、A/B测试

---

### 维度 B：结构化表达（5分）

> 考察：是否有清晰框架、是否有场景举例、是否用决策树而非清单

| 分值 | 标准 |
|------|------|
| 5 | 框架清晰（定义→对比→场景→总结），有具体场景，用决策树，有数字 |
| 4 | 框架基本清晰，有场景，偶有清单式表达 |
| 3 | 框架存在，但场景不够具体，或有清单式表达 |
| 2 | 框架不清晰，缺场景，平铺清单 |
| 1 | 无框架，散乱表达 |

**结构化表达通用框架**：

```
标准答题结构（2-3分钟）：
1. 定义/是什么（30秒）
2. 为什么/核心价值（20秒）
3. 怎么做/决策树（60秒）
4. 具体场景（30秒）
5. 总结/关键差异（20秒）
```
